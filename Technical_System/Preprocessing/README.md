# Preprocessing Tools for Multi-Modality Medical Image Fusion
## 📌 Overview
Preprocessing is a key step in the MMIF workflow, which directly affects the quality of fusion results. The preprocessing of multi-modal medical images mainly includes image registration, intensity normalization, noise reduction, skull stripping, ROI cropping, etc. This section sorts out the commonly used preprocessing tools in the MMIF field, with detailed information such as tool functions, applicable modalities, official links, and open-source licenses.

## 📑 Curated List of Preprocessing Tools
| Tool Full Name | Core Functions | Applicable Modalities & Scenarios | Official Link | Open-Source License |
|-----------------|-----------------|------------------------------------|---------------|---------------------|
| ITK-SNAP | A medical image processing tool that integrates manual annotation, semi-automatic segmentation, image registration, and 3D visualization. It is widely used in the preprocessing of 3D medical images. | All medical imaging modalities (MRI/CT/PET/SPECT), especially for 3D volume data annotation and registration. | [Official Link](http://www.itksnap.org/pmwiki/pmwiki.php) | GNU GPLv3 |
| 3D Slicer | A free and open-source medical image computing and visualization platform, which integrates a large number of preprocessing modules such as rigid/non-rigid registration, intensity normalization, noise reduction, and skull stripping. | All medical imaging modalities, especially for 3D multi-modal image preprocessing and fusion visualization. | [Official Link](https://www.slicer.org/) | BSD-style |
| MONAI | A PyTorch-based open-source framework for medical image deep learning, which provides a large number of preprocessing APIs for medical images, including intensity normalization, data augmentation, spatial transformation, etc. | All medical imaging modalities, supporting 2D/3D data preprocessing for deep learning-based MMIF. | [Official Link](https://monai.io/) | Apache 2.0 |
| Nibabel | A Python library for reading and writing neuroimaging data, which supports common medical image formats such as NIfTI, DICOM, and is widely used for loading and preprocessing of MRI/CT images. | MRI/CT/PET neuroimaging data, format conversion, basic preprocessing. | [Official Link](https://nipy.org/nibabel/) | MIT |
| Pydicom | A Python library for reading and writing DICOM files, which is the standard tool for parsing clinical medical image data, and supports the preprocessing of DICOM format images from all mainstream imaging equipment. | All medical imaging modalities in DICOM format, clinical data parsing and preprocessing. | [Official Link](https://pydicom.github.io/pydicom/stable/) | MIT |
| FSL (FMRIB Software Library) | A comprehensive library for medical image analysis, which provides professional preprocessing tools such as skull stripping, linear/non-linear registration, and intensity normalization for brain MRI images. | Brain MRI/CT/PET images, neuroimaging preprocessing. | [Official Link](https://fsl.fmrib.ox.ac.uk/fsl/fslwiki) | Non-commercial use |
| ANTs (Advanced Normalization Tools) | A high-performance medical image registration toolbox, which supports rigid, affine, and diffeomorphic non-rigid registration of multi-modal medical images, and is widely used in the preprocessing of multi-modal brain images. | Multi-modal brain MRI/CT images, high-precision image registration. | [Official Link](http://stnava.github.io/ANTs/) | Apache 2.0 |
